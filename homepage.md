## 4. Project-Driven Learning Loop

1. **Pick a small ambitious project**: e.g., CPU, OS, hardware accelerator, robotics platform, distributed system  
2. **Learn what you need**: Only study lectures/resources relevant to gaps discovered  
3. **Implement & experiment**: FPGA, software simulation, embedded device, or distributed setup  
4. **Iterate & expand**: Add complexity, solve real problems, test with users  

> Focus: “Learn while building.” Knowledge compounds much faster than passive watching.

---

## 5. High-Value Student Project Ideas

- Build a **CPU from scratch** (FPGA or simulator)  
- Write a **small operating system**  
- Create a **hardware accelerator** (e.g., matrix multiplication, AI inference)  
- Build a **distributed system** (key-value store, scheduler)  
- Prototype a **real-world sensing or robotics system**  

> Projects are stepping stones toward startup-ready skills and insights.

---

## 6. Mindset & Habits

- **Avoid knowledge hoarding**: Don’t try to learn everything before building  
- **Curiosity loop**: Watch → Experiment → Learn just enough → Apply → Repeat  
- **Focus on real problems**: Ask “What is expensive or painful in the real world?”  
- **Iterate with users**: Turn projects into prototypes that people care about
